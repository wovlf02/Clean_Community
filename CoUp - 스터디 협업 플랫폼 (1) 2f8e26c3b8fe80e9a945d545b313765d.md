### 03. 개발 중 이슈

<aside>

 **⏲ 효율성 개선**

1. API 연결을 src/app/api 폴더 내에 도메인 별로 분리하여 한 폴더에서 모두 관리할 수 있도록 파일 구조 설계
2. Next.js의 장점인 SSR과 CSR을 활용하기 위해 컴포넌트 단위로 모듈을 나누고 각 모듈마다 효율적인 방식으로 렌더링되도록 구현
3. 도메인별로 폴더를 나누고 컴포넌트 단위로 파일을 모두 분리하여 한 파일에 작성된 코드가 많아지지 않도록 코드 작성
4. jsx 파일과 css 파일을 분리하여 개발 및 유지보수 비용 최적화
5. CoUp Repository에서는 문서가 복잡한 구조로 되어있어 Cooperate_Up Repository를 하나 더 생성한 후 불필요한 문서 삭제 및 구조를 정리함
6. Next.js는 HTTP/REST API 전담, 소켓 통신 처리를 위해 Node.js 기반 시그널링 서버 독립 운영
</aside>
